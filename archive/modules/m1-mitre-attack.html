Module 1

MITRE ATT&CK — Le cadre universel

J1 · 09h30—11h00 · 1h30 · Présentation + Démo live

Le référentiel MITRE ATT&CK est le standard de fait pour décrire les comportements adverses. Adopté par le COMCYBER, l'ANSSI et l'OTAN, il sert de langage commun entre red team, blue team, SOC et RSSI. Ce module pose le socle qui structurera toute la formation.

1 Qu'est-ce que MITRE ATT&CK ?

Origine et philosophie

MITRE Corporation (à but non lucratif, fondée en 1958) développe ATT&CK depuis 2013. Le framework décrit comment les attaquants opèrent, pas les outils qu'ils utilisent. C'est un modèle basé sur l'observation de groupes APT réels.

  • ATT&CK = Adversarial Tactics, Techniques & Common Knowledge
  • Plus de 200 techniques et 400+ sub-techniques
  • Documente des groupes (APT28, Lazarus, FIN7...), des logiciels (Cobalt Strike, Mimikatz...) et des mitigations

Les 3 matrices

🌐

Enterprise

2 Structure : Tactiques → Techniques → Sub-techniques

Hiérarchie

14 Tactiques → 200+ Techniques → 400+ Sub-techniques → Procédures
  • Tactique = l'objectif de l'attaquant (ex: Initial Access, Credential Access)
  • Technique = comment il y parvient (ex: T1190 Exploit Public-Facing Application)
  • Sub-technique = variante plus précise (ex: T1505.003 Web Shell)
  • Procédure = implémentation spécifique par un groupe APT

Les 14 tactiques Enterprise

Reconnaissance → Resource Development → Initial Access → Execution → Persistence → Privilege Escalation → Defense Evasion → Credential Access → Discovery → Lateral Movement → Collection → Command and Control → Exfiltration → Impact

Adoption institutionnelle

🇦🇷🇱🇪 OTAN / NATO
🇫🇷 DGA

  • COMCYBER : ATT&CK est intégré dans la doctrine de Lutte Informatique Défensive (LID). Utilisé pour modéliser les menaces, qualifier les incidents, et planifier les exercices.
  • ANSSI : Référentiel pour les PASSI (Prestataires d'Audit de Sécurité des Systèmes d'Information). Utilisé dans les guides d'hygiène et la qualification des produits de sécurité.
  • OTAN : Cadre commun de Threat Intelligence partagé entre les pays membres. Permet l'interopérabilité des cellules de cyberdéfense.
  • DGA : Intègre ATT&CK dans les cahiers des charges des systèmes d'armes pour les exigences de cybersécurité.

Cas d'usage concrets

  • Threat Modeling : cartographier la surface d'attaque d'un système avant déploiement
  • Gap Analysis : identifier les angles morts de détection d'un SOC
  • Red Teaming : planifier les TTPs à tester lors d'un exercice
  • Reporting : produire des rapports structurés et comparables
  • CTI (Cyber Threat Intelligence) : suivre l'évolution des groupes APT

4 Lien avec NIS2

ATT&CK, outil de conformité NIS2

La directive NIS2 (Directive (UE) 2022/2555) impose aux entités européennes des obligations renforcées en matière de cybersécurité : modélisation des menaces (threat modeling), détection des incidents et notification dans un délai de 24 heures.

MITRE ATT&CK est le framework de référence pour répondre à ces exigences. Il permet notamment de satisfaire :

  • Article 21 — Mesures de gestion des risques : ATT&CK fournit la base de connaissance pour le threat modeling, la gap analysis et l'évaluation continue de la posture de sécurité.
  • Article 23 — Obligations de signalement : les techniques ATT&CK permettent de qualifier et structurer les rapports d'incidents de manière standardisée et interopérable.

En France, NIS2 est transposée sous l'égide de l'ANSSI et s'applique aux Entités Essentielles (EE) et aux Entités Importantes (EI). Le COMCYBER utilise ATT&CK pour aligner la doctrine militaire française (LID — Lutte Informatique Défensive) avec les exigences NIS2, assurant une cohérence entre les dimensions civile et militaire de la cyberdéfense nationale.

6 ATT&CK comme outil de reporting

Ce qu'on attend en fin de formation (J4)

  • Matrice de couverture : heat map des techniques testées vs détectées
  • Gap analysis : techniques non couvertes → recommandations
  • Rapport exécutif : synthèse lisible par un RSSI avec visualisation ATT&CK
  • Rapport technique : chaque vulnérabilité taggée TXXXX, avec preuve d'exploitation et remédiation

Outil : ATT&CK Navigator

Installer et utiliser l'ATT&CK Navigator pour créer des heat maps :

# Cloner le dépôt officiel
git clone https://github.com/mitre-attack/attack-navigator

# Lancer avec Docker
cd attack-navigator
docker build -t attack_navigator .
docker run -p 4200:4200 attack_navigator

# Puis ouvrir http://localhost:4200
